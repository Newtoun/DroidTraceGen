# GravadorDeAcoes — Backend Server

---

## English

### Overview

This backend server is part of the **GravadorDeAcoes** toolchain — an Android tool that records UI actions via the Accessibility API and feeds them into an LLM-based test case generation pipeline.

The example test cases provided in this repository were used as part of an **experimental evaluation**, where manually written test cases (ground truth) were compared against test cases automatically generated by the pipeline.

### Folder Structure

```
BackendSocket/
├── app.py               # TCP server — receives logs from the Android app
├── prompt.py            # Sends logs to the LLM and generates TDD test cases
├── dockerfile           # Docker image definition
├── bdd gemini pro/      # Reference test cases used in the experimental evaluation
├── analysis_data/       # 5 synthetic TC logs captured from the Android app (log_N.txt)
└── tc_result/           # TDD test cases generated from the captured steps (log_N_testcase.txt)
```

### How It Works

1. The Android app records UI actions and sends them as a log via TCP
2. The server receives the log and saves it to `analysis_data/`
3. `prompt.py` reads the latest log and sends it to the **LLaMA 3.1 70B** model (via Ollama)
4. The generated TDD test case is saved to `tc_result/`

The Android app **automatically discovers** the server on the local network using TCP subnet scanning — no IP configuration required on either side.

### Requirements

- Python 3.8+ **or** Docker
- [Ollama](https://ollama.com) running locally with `llama3.1:70b` pulled

### Running with Docker

Build:
```
docker build -t gravador-server .
```

Run (Linux/macOS):
```
docker run -d \
  --name gravador-server \
  -p 12345:12345 \
  -v "$(pwd)/analysis_data:/app/analysis_data" \
  gravador-server
```

Run (Windows PowerShell):
```
docker run -d `
  --name gravador-server `
  -p 12345:12345 `
  -v "${PWD}/analysis_data:/app/analysis_data" `
  gravador-server
```

### Running with Python

```
pip install requests
python app.py
```

### Generating Test Cases

```
python prompt.py
```

Output is saved to `tc_result/log_N_testcase.txt` in structured TDD format.

### LLM Model

| Parameter | Value              |
|-----------|--------------------|
| Model     | LLaMA 3.1 70B      |
| Provider  | Ollama (local)     |
| Endpoint  | http://127.0.0.1:11435/api/generate |

### Port

| Port  | Protocol | Purpose                                     |
|-------|----------|---------------------------------------------|
| 12345 | TCP      | Server discovery handshake + log reception  |

## Português

### Visão Geral

Este servidor backend faz parte da toolchain **GravadorDeAcoes** — uma ferramenta Android que grava ações de interface via Accessibility API e as alimenta em um pipeline de geração de casos de teste baseado em LLM.

Os casos de teste de exemplo fornecidos neste repositório foram utilizados como parte de uma **avaliação experimental**, na qual casos de teste escritos manualmente (ground truth) foram comparados com casos de teste gerados automaticamente pelo pipeline.

### Estrutura de Pastas

```
BackendSocket/
├── app.py               # Servidor TCP — recebe logs do app Android
├── prompt.py            # Envia logs ao LLM e gera casos de teste TDD
├── dockerfile           # Definição da imagem Docker
├── bdd gemini pro/      # Casos de teste de referência utilizados na avaliação experimental
├── analysis_data/       # 5 logs de TC sintético capturados pelo app Android (log_N.txt)
└── tc_result/           # Casos de teste TDD gerados a partir dos passos capturados (log_N_testcase.txt)
```

### Como Funciona

1. O app Android grava ações de UI e as envia como log via TCP
2. O servidor recebe o log e o salva em `analysis_data/`
3. `prompt.py` lê o log mais recente e o envia ao modelo **LLaMA 3.1 70B** (via Ollama)
4. O caso de teste TDD gerado é salvo em `tc_result/`

O app Android **descobre automaticamente** o servidor na rede local por varredura TCP de sub-rede — nenhuma configuração de IP é necessária em nenhum dos lados.

### Requisitos

- Python 3.8+ **ou** Docker
- [Ollama](https://ollama.com) rodando localmente com o modelo `llama3.1:70b` baixado
